def validate_triangle(a, b, c):
    if a <= 0 or b <= 0 or c <= 0:
        raise ValueError("Sides must be positive")

    if not (a + b > c and b + c > a and c + a > b):
        raise ValueError("Invalid triangle (triangle inequality failed)")

👉 This alone catches:
	•	user entering angle as side (60)
	•	bad coordinate combos
	•	wrong conversions
	•	SSA ambiguity (invalid branch)
